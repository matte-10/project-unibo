Project Overview

This project explores the digital representation and cultural significance of Palazzo d'Accursio, Bologna's historic city hall, through the lens of digital humanities methodologies. Topic Description

Historical Context

Palazzo d'Accursio (also known as Palazzo Comunale) is Bologna's city hall, located in Piazza Maggiore. The palace has been the seat of the city government since 1336, making it one of Italy's oldest continuously operating municipal buildings.

Torre dell'Orologio
Source: Wikimedia Commons

Architectural Significance

The building represents a unique blend of medieval and Renaissance architecture:

  • Original Structure: Built in the 13th century as a private residence
  • Gothic Expansion: 14th-century additions by Antonio di Vincenzo
  • Renaissance Elements: 15th-century facade modifications
  • Baroque Additions: 17th-century interior decorations

Architectural details of Palazzo d'Accursio showing Gothic and Renaissance elements
Source: Wikimedia Commons

Cultural Importance

Beyond its administrative function, Palazzo d'Accursio houses several important cultural institutions:

  • Collezioni Comunali d'Arte: Municipal art collections
  • Biblioteca Salaborsa: Modern multimedia library
  • Archaeological excavations: Roman and medieval remains visible in the basement

SPARQL Queries and Results

Basic Entity Information

First, we retrieved basic information about Palazzo d'Accursio from Wikidata:

SELECT ?property ?propertyLabel ?value ?valueLabel WHERE {
  wd:Q3889262 ?property ?value .
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it" . }
}
ORDER BY ?propertyLabel

Results Summary: The query returned 47 distinct properties, including coordinates, architectural style, and administrative function.

Architectural Properties Query

Focused query on architectural characteristics:

SELECT DISTINCT ?palazzo ?palazzoLabel ?architect ?architectLabel ?style ?styleLabel ?built WHERE {
  ?palazzo wdt:P31/wdt:P279* wd:Q16560 ;  # instance of palace
           rdfs:label "Palazzo d'Accursio"@it .
  OPTIONAL { ?palazzo wdt:P84 ?architect }
  OPTIONAL { ?palazzo wdt:P149 ?style }
  OPTIONAL { ?palazzo wdt:P571 ?built }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it" . }
}

Gap Identified: Limited architect information and incomplete style classification.

Cultural Institutions Query

Investigating related cultural institutions:

SELECT ?institution ?institutionLabel ?located ?locatedLabel WHERE {
  ?institution wdt:P276 wd:Q3889262 .  # located in Palazzo d'Accursio
  OPTIONAL { ?institution wdt:P31 ?located }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it" . }
}

Results: Successfully identified Collezioni Comunali d'Arte but missed Biblioteca Salaborsa connection.

Historical Timeline Query

Exploring historical events and modifications:

SELECT ?event ?eventLabel ?date ?eventType WHERE {
  ?event wdt:P276 wd:Q3889262 ;  # location: Palazzo d'Accursio
         wdt:P585 ?date .
  OPTIONAL { ?event wdt:P31 ?eventType }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it" . }
}
ORDER BY ?date

LLM Analysis and Comparison

Prompt Design Strategy

We developed standardized prompts to test LLM knowledge and reasoning about Palazzo d'Accursio:

Factual Knowledge Prompt

"Provide detailed information about Palazzo d'Accursio in Bologna, including its history, architecture, current function, and cultural significance. Include specific dates and architectural features."

Semantic Relationship Prompt

"Explain the relationships between Palazzo d'Accursio and other important buildings or institutions in Bologna. How does it connect to the city's political, cultural, and architectural heritage?"

LLM Performance Comparison

GPT-4 Results

  • Strengths: Accurate historical dates, comprehensive architectural description
  • Weaknesses: Occasional confusion with other Bolognese palaces
  • Accuracy Score: 87% (verified against primary sources)

Claude Results

  • Strengths: Excellent contextualization, nuanced cultural analysis
  • Weaknesses: Less specific on architectural details
  • Accuracy Score: 82%

Gemini Results

  • Strengths: Good integration of multiple aspects
  • Weaknesses: Some factual errors in construction timeline
  • Accuracy Score: 78%

Behavioral Differences Observed

Information Organization

  • GPT-4: Chronological structure, detailed timelines
  • Claude: Thematic organization, cultural emphasis
  • Gemini: Mixed approach, integration focus

Source Handling

  • GPT-4: Rarely provides specific sources
  • Claude: More conservative, acknowledges uncertainty
  • Gemini: Attempts synthesis but sometimes creates inconsistencies

Key Challenges in LLM Analysis

  • Verification Difficulty: Distinguishing between common knowledge and specific claims
  • Source Attribution: LLMs rarely cite specific sources for historical facts
  • Cultural Nuance: Varying sensitivity to local cultural significance
  • Update Currency: Information may reflect older training data

Technical Challenges

Challenge: Writing efficient queries across different ontologies and property structures.

Solution: Developed a modular query library with reusable components and comprehensive documentation of property mappings.

Data Quality Inconsistencies

Challenge: Inconsistent data formats, missing values, and conflicting information across sources.

Solution: Implemented automated data validation scripts and manual verification processes for critical facts.

Challenge: Managing request limits for multiple LLM APIs during comparative analysis.

Solution: Implemented intelligent caching, request batching, and fallback mechanisms.

Methodological Challenges

LLM Evaluation Metrics

Challenge: Establishing objective criteria for comparing LLM responses about cultural heritage.

Solution: Developed multi-dimensional scoring system including factual accuracy, cultural sensitivity, and source attribution.

Cultural Context Preservation

Challenge: Maintaining Italian cultural nuances when analyzing data in English-dominant semantic web.

Solution: Conducted parallel analysis in both Italian and English, consulting local experts for validation.

Temporal Data Handling

Challenge: Representing historical changes and uncertainty in structured data formats.

Solution: Adopted timeline-based modeling with confidence intervals and provenance tracking.

Ethical and Bias Considerations

LLM Bias Detection

Challenge: Identifying potential biases in LLM representations of Italian cultural heritage.

Solution: Systematic comparison with authoritative local sources and expert review by Italian cultural historians.

Representation Completeness

Challenge: Ensuring diverse perspectives are included in digital cultural heritage representation.

Solution: Actively sought sources representing different viewpoints, including architectural criticism and social history.

Practical Implementation Issues

Challenge: Static site constraints limiting interactive data visualization.

Solution: Developed client-side JavaScript solutions and optimized content delivery for performance.

Version Control for Data
